Gatling Performance Framework

A Gatling-based performance framework (Java) to create repeatable, CI-friendly load tests for APIs and web endpoints. Designed for realistic user scenarios, configurable test data, pass/fail assertions and rich HTML reporting.

  • Realistic load: simulate user journeys with injected traffic (ramp, constant, spike profiles).
  • SLA assertions: fail CI on breaches (error rate, p95/p99, mean response time).
  • Reports & artifacts: rich HTML reports generated under target/gatling/ for sharing and trend analysis.
  • CI friendly: run via mvn gatling:test, upload reports as artifacts and gate PRs with lightweight smoke runs.
  • Scalable execution: run from containers, distributed runners or dedicated load nodes for large-scale tests.
  • Observability: integrate with Prometheus/Grafana or APM tools to correlate app metrics with load.
  • Reusable assets: scenario templates, shared feeders and assertions to maintain consistency across tests.

Quick start: add the Gatling Maven plugin, place simulations in src/gatling/simulations, provide feeders in src/gatling/resources, then run mvn gatling:test.

Technology Used : Java , Gradle . BDD ,TestNg, Gatling
